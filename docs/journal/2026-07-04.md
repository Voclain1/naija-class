## Work log

### PaystackService constructor hardening

**Problem**: `PaystackService` threw in its constructor when `PAYSTACK_SECRET_KEY`
was missing. A missing env var on staging was crashing the entire NestJS process
at startup — taking down all modules, not just payments.

**Fix** (`apps/api/src/common/paystack/paystack.service.ts`):

- Constructor now logs `logger.warn(...)` and sets `secretKey = key ?? ''`
  instead of throwing.
- `initializeTransaction()` and `verifyTransaction()` each guard at the top:
  ```
  if (!this.secretKey) throw new Error('PAYSTACK_SECRET_KEY is not configured on this server')
  ```
- `verifyWebhookSignature` unchanged — an HMAC computed with an empty key
  produces a 64-byte hex string that never matches a real Paystack signature,
  so it already fails closed.

Result: the app starts successfully without the key; only actual Paystack API
calls fail (with a clear error), not the whole process.

### Test update (`apps/api/src/common/paystack/paystack.service.spec.ts`)

The old constructor test asserted `toThrow("PAYSTACK_SECRET_KEY is not set")` —
now a false negative. Updated the describe block:

| Old | New |
|---|---|
| `throws at construction time if PAYSTACK_SECRET_KEY is not set` | `logs a warning if PAYSTACK_SECRET_KEY is not set` — asserts `not.toThrow()`, suppresses console output via `vi.spyOn(console, 'warn')` |
| _(none)_ | `throws at call time if PAYSTACK_SECRET_KEY is not set` — constructs a no-key service, asserts `initializeTransaction(...)` rejects with `'PAYSTACK_SECRET_KEY is not configured'` |

Net: one old test replaced by two new ones (`+1` to suite total). Added `vi`
to the vitest import.
