## Work log

### Refunds & reversals

**Migrations** (`packages/db/prisma/migrations/`):

- `Refund` model + `RefundStatus` enum (`REQUESTED | PROCESSED | FAILED`), with
  ENABLE/FORCE RLS and a `tenant_isolation` policy, matching the pattern used
  by every other Phase 3 finance table.
- Second, idempotent migration grants `payment.refund` to the admin role
  (RBAC data migration, safe to re-run).

**`RefundsService`** (`apps/api/src/modules/payments/refunds.service.ts`):

- Full-reversal-only: `POST /refunds` rejects any amount that doesn't equal
  `payment.amount` with `PARTIAL_REFUND_NOT_SUPPORTED`.
- Two-path design:
  - Manual payment → marks the payment `REVERSED` and writes a `PROCESSED`
    `Refund` row immediately.
  - Paystack payment → calls `PaystackService.refundTransaction` (Paystack
    `POST /refund`) **before** any DB write. On API rejection, writes a
    `FAILED` `Refund` row as an audit trail and makes no other mutation.
- Recompute pattern: `invoice.totalPaid` is recomputed as
  `aggregate(SUCCESS payments)` — `REVERSED` payments simply drop out, no
  separate "reversal" bookkeeping needed.
- Status override: `newTotalPaid === 0 ? REFUNDED : computeInvoiceStatus(...)`
  guarantees a fully-reversed invoice always lands on `REFUNDED`, never
  `ISSUED`/`OVERDUE`.
- `PaymentPlanService.recomputeInstallmentsPaid` called on every reversal to
  un-mark installments that are no longer covered.
- Every `PROCESSED` refund writes an `audit_logs` row with action
  `refund.create`.

**Bug fix in `computeInvoiceStatus`**: extended with an optional `dueDate`
param to preserve `OVERDUE` across a partial reversal. This also fixed a
pre-existing bug — an `OVERDUE` invoice with a partial payment reversed was
incorrectly falling back to `PARTIALLY_PAID` instead of staying `OVERDUE`.

**`RefundsController`** (`apps/api/src/modules/payments/refunds.controller.ts`):

```
POST /refunds   @Permissions('payment.refund')
```

`payment.refund` is admin/owner only — bursar is explicitly excluded, unlike
most other finance permissions.

**Tests**: 10 new integration tests in `refunds.service.spec.ts`;
`permissions-coverage.spec.ts` extended to 74 total.

**Web UI** (`apps/web/src/app/(admin)/finance/invoices/[id]/page.tsx`):
Reverse button + reason modal on the invoice detail page, wired to the new
`payments-api.ts` client function.
