## What shipped

### Slice 6 invoice detail — two display fixes

**Fix 1 — uncapped discount column.**
The discount column in `apps/web/src/app/(admin)/finance/invoices/[id]/page.tsx` was
summing the raw `discountsApplied` amounts, which could exceed the item's gross amount
when multiple rules stacked. Fixed by deriving the display value as
`item.amount - item.netAmount` (netAmount is already correctly capped at the service layer
in `buildSnapshot()`). The tooltip still shows individual rule amounts for transparency.

**Fix 2 — UUID fields showing as raw IDs.**
Student and term were displaying as opaque UUIDs because `InvoiceDto` only carries
`studentId` and `termId`. Fixed by firing two parallel non-fatal sub-fetches after the
invoice loads: `getStudent(studentId)` for `firstName + lastName (admissionNumber)`,
and `listTerms(academicYearId)` filtered to the matching term id for the term name.
Both fall back to UUID display on failure — the invoice page stays functional even if
the secondary fetches 404.

## Plan-first locked — Phase 3 Slice 7 (manual payment recording + receipts)

Full decisions written into `docs/modules/phase-3.md §16`. Summary:

| Decision | Locked choice | Rationale |
|---|---|---|
| Overpayment | **REJECT** (`ConflictError`) | Data-entry error in Nigerian school context; PAID = fully settled, no credit balance |
| `totalPaid` update | **RECOMPUTE** from `SUM(amount) WHERE status = 'SUCCESS'` | Idempotent; same path reusable by Paystack (slice 8) and reversals (slice 11) |
| Status transition | **Pure fn** `computeInvoiceStatus(totalPaid, totalDue)` in service layer | Not a DB trigger; testable in isolation; CANCELLED/REFUNDED are terminal |
| Receipt format | **HTML on R2** (not PDF) | No Puppeteer dependency needed; bursar prints from browser; fast-follow to PDF is a content-type swap |
| Manual payment status default | **SUCCESS** | Confirmed-in-hand at record time; PENDING reserved for Paystack initiation |
| `paidAt` | **Any valid ISO datetime** (no backdate limit) | Late entry is normal; audit log preserves `createdAt` gap |
| Receipt number | **`RCP-<paymentId.slice(0,8).toUpperCase()>`** | UUID-derived, unique, no sequence table |
| `StorageObjectKey` | Add `{ kind: "payment-receipt"; paymentId: string }` | Path: `schools/${schoolId}/receipts/${paymentId}.html` |

New permissions: `payment.read`, `payment.record` → added to `PHASE_3_PERMISSIONS`.
`payment.refund` deferred to slice 11.
